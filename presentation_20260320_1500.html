❓ Team Question

Using Claude Projects for Specs & Tech Stack

"Will we use Claude Projects to create Requirement and Design Specs and dictate a specific Technology stack in Claude Code?"

✅ Yes — Here's How

▶
1

Claude Projects → Spec Authoring

Use Claude Projects (claude.ai web) for collaborative spec writing. Upload existing docs, reference materials, and iterate on requirements with the team. Export finalized specs as markdown files into your repo.

▶
2

CLAUDE.md → Tech Stack Enforcement

Define your tech stack in CLAUDE.md at the project root. Claude Code reads this file automatically and enforces these constraints in every interaction:

  • Language, framework, and library versions
  • Architecture patterns (clean arch, hexagonal, etc.)
  • Database choices, ORM preferences
  • "Never use X" / "Always use Y" rules
▶
3

Spec → Code Pipeline

With specs in the repo and rules in CLAUDE.md, you can give Claude Code high-level instructions like:

  • "Implement the auth module per docs/auth-spec.md"
  • "Create the database schema from docs/data-model.md"
  • "Build the API endpoints listed in docs/api-contract.yaml"

Claude Code will read the spec and generate code that follows your stack constraints.

🔄 The Flow

1 Draft specs in Claude Projects (web UI)
2 Export as markdown → docs/ folder in repo
3 Define stack rules in CLAUDE.md
4 Claude Code reads both & generates compliant code
5 Iterate: update specs → Claude Code adapts
Key Insight: Claude Projects is great for collaborative authoring; Claude Code is great for turning those specs into working code. Use both together.
❓ Team Question

Multi-Module Team Development

"How will different team members develop different modules but still connect/use functionalities developed by others?"

🏗️ Shared Codebase, Individual Branches

  • Monorepo or modular repo — everyone works in the same Git repo
  • Each developer has their own feature branch
  • Claude Code reads the entire codebase including other members' modules
  • When Dev B needs to call Dev A's auth service, Claude already knows the interface

🔗 Interface Contracts

  • Define shared interfaces/types in a common directory
  • API contracts via OpenAPI specs in the repo
  • Claude Code enforces type safety across module boundaries
  • Catches integration issues before they reach CI

🔄 Pull & Rebase Workflow

  • Regular pulls from main keep modules in sync
  • Claude Code helps resolve merge conflicts intelligently
  • Ask: "What changed in the auth module since I last pulled?"

❓ Team Question — Most Important

Long-Term Maintenance & Evolution

"Having the first version developed with Claude is fine. It's more important to understand how we use Claude to keep adding/changing functionalities, fix bugs etc. after one or two years of the first release."

🎯 This is where Claude Code truly shines

🐛 Bug Fixes — Deep Root Cause Analysis

Describe the bug in plain English. Claude reads your entire codebase (up to 1M tokens of context), traces execution paths, identifies the root cause, explains it, and proposes a targeted fix — even in code written years ago by someone else.

✨ Feature Additions — Context-Aware

Claude Code reads existing patterns, conventions, and architecture. New features are built consistently with the rest of the codebase — not in a different style every time a new dev touches it.

🔄 Refactoring — Safe, Cross-Codebase

Need to rename a service, change a data model, or swap out a library? Claude traces all usages, updates every reference, and runs tests to verify nothing breaks.

📦 Dependency Upgrades

Major version upgrades, security patches, breaking API changes — Claude reads migration guides, identifies breaking changes in your code, and applies the fix across the codebase.

🧑‍🏫 Onboarding New Team Members

New developers ask Claude: "How does the payment flow work?" and get an accurate, up-to-date explanation derived from the actual code, not outdated docs.

📖 Living Documentation

CLAUDE.md and project docs evolve with the code. Claude helps maintain ADRs, API docs, and changelogs as the project grows.
